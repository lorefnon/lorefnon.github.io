API exploration on the command line

httpie is a great utility that provides a user-friendly CLI for making HTTP calls. Since I started using it a year ago, it has become a part of my everyday toolbelt.

In the developer community, utilities like postman have become quite popular. Besides providing a solution for quickly testing out HTTP APIs, such tools also facilitate managing of collections. One great thing about postman collections is that they can be templated through variables. While the UI for managing these variables is a bit clunky, it provides a simple way to use the sample collection for multiple environments.

This post outlines a simple solution to collection management by composing together a few more open source utilities.

The directory structure of our collection will look like following (We are using the shopify API as an example): 

api-collection
|_ order-admin-api
|   |_ variables.yaml
|   |_ invocations
|        |_ fetch-order-by-id.sh
|        |_ fetch-order-by-date-range.sh
|_ fulfillment-admin-api
|   |_ variables.yaml
|   |_ invocations
|        |_ fetch-fulfillments-by-order-id.sh
|_ varaibles.yaml
